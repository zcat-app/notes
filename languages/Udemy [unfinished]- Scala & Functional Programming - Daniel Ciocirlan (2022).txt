Values
- Kotlin like declaration & initialization (no semicolons needed - optional):
- 'val' ~ like Kotlin - immutable, similar to Java 'final'
        - syntax: val name[:Type] = value
        - type is optional - compiler infer data type ~ same as Kotlin

    
            val x:Int = 42   //same as val x = 42
            println(x)

Variables
- Kotlin like declaration & initialization (no semicolons needed - optional):
- 'var' ~ like Kotlin, can re-assigned, used in functional prgramming for 'side effects'  
        - syntax: var name[:Type] = value
        - type is optional - compiler infer data type ~ same as Kotlin

            var x:Int = 42   //same as val x = 42
            println(x)

- vals preffereable over vars (no side effect)

Expressions
- instruction - say to do something -> store, send, ...
- expression  - evaluate something -> provides result value
- everything in scala is expresion (except val/var declaration) - yeah, even reassigning and loops

- compiler can infer type of expression (if assigned)
- num: + - * / %
  bit: & | ^ << >> >>>  - right shift with zero extension 
  logical: == != > >= < <= ! && ||
  assign: += -= *= /= 
         -> side effects 

- if expression ~ kinda ternary operator
    val result = if (condition) resutTrue else resultFalse
