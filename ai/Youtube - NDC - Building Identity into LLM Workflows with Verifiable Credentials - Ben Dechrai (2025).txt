Prompt injection
- growing threat for Agents
- can be text in email, memo in bank transfer etc.
- guardrails might not always work - manipulation
- if maintaining communication with API - usually bearer token
- bearer token is stored in agent context - does not verify the owner
- once token hijacked - disabled on expiration, or recreating
- problem in multi agentic world - agent that called malicious API might not be the one infected - hard to track

Solution - verifiable credentials
- JSON with roles, credential object, issuer, proof type
- credential signed by issuer (any authority, might be employer, independent authority, etc)
- standard in w3.org - https://www.w3.org/TR/vc-data-model-2.0/
- example (from wiki: https://en.wikipedia.org/wiki/Verifiable_credentials):

- verification with verifiable presentation (more info: openID - https://openid.net/specs/openid-4-verifiable-presentations-1_0.html)
   - JSON with wrapped credentials
   - has one-time use challenge as nonce
   - signed by credentials holder private key (have to have private key)
   - did ~ decentralized identity
         - public key - simple (no server needed), instant verification, no key rotation
         - web - bound to domain, allows key rotations, fast (depend on server how fast it will answer)
         - ioa - BTC wallet- legacy, fading support, key rotation possible but longer verification time (minutes)

- basically oauth like verification, key never leaves, fulfilling challenge as openID thing
   wallet         agent                auth          server
                  - asks for data
                                                     - auth needed
                                       - wants presentation
                                       - send challenge
                  - pass challenge to sign
   - sign presentation
                  - send to auth as JWT
                                       - verifies, respond with short lived JWT
                  - send request to fetch data from server
                                                     - response with data
