- control tokens - labeling the system prompt
                 - model specific
                 - pure text, parsed on fly
                 - using then as user input - changing system prompt

- prompt templates - pre-prepared prompts with 'ensured' good output
                   - used for agents or to enhance user input - danger that placeholders can be used to 'inject' prompt
                   - e.g. start with closing control tokens like </userInput> [here define system config, etc..]

- guardrails
   - LLM judge - another LLM evaluating malicious action, prompt injection attempt, etc
               - high latency
               - high cost
               - same understanding as downstream target (model we pass it to)
               - prompting, fine tuning
               - possible to prompt inject guardrail - wrapping prompt injection into prompt injection
                                                     - e.g. manipulation threshold to let pass 'smelly' input
                                                     - idea to rank result - if over threshold - not pass, else OK

   - Classification model - model to rank prompt as safe or malicious
                          - lower cost
                          - low latency
                          - different understanding as downstream target
                          - fine tuning only
                          - uses tokenizer - converts text to numbers to pass to model
                                           - different types: BPE
                                                              Wordpiece
                                                              Unigram
                                           - different processing of text - different evaluation result
                                                                          - limited understanding of input against LLM judge
